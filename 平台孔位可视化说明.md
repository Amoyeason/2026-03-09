# 平台孔位可视化功能说明

## 📋 问题背景

### 应用场景
1. **加工对象**：大型薄壁壁板，加工时易变形
2. **支撑方案**：柔性平台（5×24孔位阵列）+ POGO柱
3. **定位问题**：
   - 姿态定位：薄板在空间中的姿态（投影面积最大）
   - 位置定位：薄板与平台孔位的位置关系

### 核心需求
根据 `check_data` 的优化结果，反推平台的真实位置，并可视化：
- **平台孔位的真实坐标**：5×24个孔在平台自身坐标系中的位置
- **平台孔位的投影坐标**：这些孔投影到薄板表面的位置
- **对比显示**：同时显示两者，便于理解定位关系

## 🔧 代码修改内容

### 1. 新增函数

#### `compute_platform_real_coordinates()`
**功能**：计算平台孔位在真实场景下的坐标

**参数**：
- `grid2d_all`: 投影到薄板上的网格点 (N, 2)，PCA对齐坐标系
- `rows`: 平台行数（默认5）
- `cols`: 平台列数（默认24）
- `spacing`: 孔位间距（默认0.25m）
- `z_platform`: 平台的Z高度（默认0.0）

**返回**：
- `platform_holes_real_3d`: 平台孔位的真实3D坐标 (120, 3)
- `platform_base_point`: 平台基准点（左下角第一个孔）的坐标 (x, y, z)

**原理**：
```python
# 平台基准点 = grid2d_all的第一个点（左下角）
platform_base_point = [grid2d_all[0, 0], grid2d_all[0, 1], 0.0]

# 每个孔位的真实坐标
x_real = platform_base_point[0] + col * spacing
y_real = platform_base_point[1] + row * spacing
z_real = 0.0  # 平台在Z=0平面
```

#### `export_platform_coordinates()`
**功能**：导出平台孔位坐标到CSV文件

**输出文件格式**：
```csv
# Platform Coordinate System Information
# Base Point (left-bottom corner, first hole): X=...m, Y=...m, Z=...m
# Grid: 5 rows x 24 cols = 120 holes
# Spacing: 0.25m
#
row,col,x_real(m),y_real(m),z_real(m)
0,0,1.234567,2.345678,0.000000
0,1,1.484567,2.345678,0.000000
...
```

### 2. 修改可视化函数 `visualize_scene()`

**新增参数**：
- `platform_holes_real_3d`: 平台孔位的真实3D坐标
- `platform_holes_projected_3d`: 平台孔位投影到薄板的3D坐标

**新增可视化元素**：
- 🔵 **蓝色点**：平台孔位的真实位置（在Z=0平面）
- 🟢 **绿色点**：平台孔位投影到薄板表面的位置
- 🔗 **青色连线**：显示真实位置与投影位置的对应关系

### 3. 主函数修改

在 `main()` 函数中添加了平台孔位计算和可视化的代码：

```python
# 计算平台孔位的真实坐标
platform_holes_real_3d, platform_base_point = compute_platform_real_coordinates(
    grid2d_all=grid2d_all,
    rows=GRID_ROWS,
    cols=GRID_COLS,
    spacing=GRID_SPACING_M,
    z_platform=0.0
)

# 平台孔位投影到薄板
platform_holes_projected_3d = lift_points_to_surface(nodes, grid2d_all, k=12)

# 导出坐标
export_platform_coordinates(...)

# 可视化
visualize_scene(..., 
                platform_holes_real_3d=platform_holes_real_3d,
                platform_holes_projected_3d=platform_holes_projected_3d)
```

## 📐 关键理解

### 平台基准点的计算
平台基准点（左下角第一个孔）的坐标就是 `grid2d_all[0]` 在PCA坐标系中的位置：
- `grid2d_all` 是按行列顺序生成的
- 第一个点 (row=0, col=0) 就是左下角
- 基准点坐标 = `[grid2d_all[0, 0], grid2d_all[0, 1], 0.0]`

### 平台孔位的真实坐标
每个孔位的真实坐标计算公式：
```
x_real = base_x + col * 0.25
y_real = base_y + row * 0.25
z_real = 0.0
```

### 投影坐标 vs 真实坐标
- **投影坐标**：孔位投影到薄板表面，Z值随薄板曲面变化
- **真实坐标**：孔位在平台平面上，Z值固定为0
- **连线**：显示两者的垂直距离，反映薄板的曲面形状

---

## ✅ 验证方法

1. **检查基准点**：确认左下角孔位坐标是否合理
2. **检查间距**：相邻孔位距离应为0.25m
3. **检查投影距离**：平均距离应在合理范围内（通常<0.1m）
4. **可视化检查**：蓝色点应在薄板下方，绿色点应在薄板表面

---

## 📝 注意事项

1. **坐标系一致性**：所有坐标都在PCA对齐坐标系中
2. **平台平面假设**：假设平台是平面（Z=0）
3. **孔位顺序**：按行优先顺序（row 0→4, col 0→23）
4. **文件编码**：CSV文件使用UTF-8编码

## 🔍 技术细节

### 平台基准点的物理意义
- **定义**：平台左下角第一个孔位的中心点
- **作用**：作为平台坐标系的原点参考
- **计算**：从优化后的网格原点直接获取

### 为什么需要区分真实坐标和投影坐标？
1. **真实坐标**：用于平台安装定位，指导硬件布置
2. **投影坐标**：用于评估支撑效果，计算覆盖率
3. **两者差异**：反映薄板的曲面形状和变形

### 坐标系统的统一性
- 平台XY方向与薄板PCA坐标系相同
- 平台Z=0，薄板Z值随曲面变化
- 这样设计便于计算和理解

## 📞 问题排查

### 如果可视化窗口不显示蓝色/绿色点
检查：
1. `platform_holes_real_3d` 是否为None
2. 数组长度是否为120
3. 坐标值是否在合理范围内

### 如果CSV文件为空或格式错误
检查：
1. `EXPORT_DIR` 目录是否存在
2. 文件写入权限
3. 编码是否为UTF-8

### 如果投影距离异常大
可能原因：
1. 薄板模型加载错误
2. PCA对齐失败
3. lift_points_to_surface的k值不合适
